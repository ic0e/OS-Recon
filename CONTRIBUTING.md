# Contributing
Thanks for your interest in OS-Recon. It's an early MVP so contributions, bug reports, and feedback are all welcome.

## Ways to help right now:

- Test the scanner and report false positives (open an issue with the platform and what it returned)
- Test the nodriver deep pry layer - it's the most fragile part
- Suggest platforms to add to the social scanner

## If you want to contribute code:

1. Fork the repo
2. Create a branch (`git checkout -b feature/your-feature`)
3. Commit your changes
4. Open a pull request with a short description of what you changed and why

Setting up locally - see the README for backend and frontend setup instructions.

For larger changes, open an issue first so we can discuss before you build it.

## AI Policy

Using AI tools while contributing is not discouraged, feel free to do so as long as you check and verify the code works. 

However, **automated AI agent PRs are not accepted** - no unsupervised bots opening 
pull requests or issues on their behalf. Every contribution should have a human 
who wrote it, reviewed it, and stands behind it.
